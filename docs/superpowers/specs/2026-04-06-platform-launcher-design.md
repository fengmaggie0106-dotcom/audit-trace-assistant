# 平台一键启动脚本设计

## 目标

为 Windows 本地开发环境提供一套统一的根目录脚本，让用户可以双击启动平台，并且能随时查看状态或停止服务，而不必手动分别进入前后端目录执行命令。

## 方案

采用根目录统一入口：

- `start-platform.ps1`
  负责静默后台启动后端与前端，写入 PID 文件，并在启动后做健康检查。
- `stop-platform.ps1`
  负责按 PID 停止服务，并清理失效 PID 文件。
- `status-platform.ps1`
  负责输出前后端当前进程状态和可访问地址。
- `start-platform.bat`
  作为双击入口，内部转调 PowerShell 脚本并使用 `ExecutionPolicy Bypass`。

## 行为约束

- 启动脚本优先复用现有进程，避免重复启动。
- 启动前会检测现有 PID 文件是否有效，并清理失效记录。
- 默认本地地址保持不变：
  - 后端：`http://127.0.0.1:8000`
  - 前端：`http://127.0.0.1:3000`
- 前端仍使用 `NEXT_PUBLIC_API_BASE=http://127.0.0.1:8000`
- 为了兼容后台静默启动，前端采用启动前自动 `npm run build`，随后后台执行 `npm run start`，而不是后台 `npm run dev`
- 脚本只负责本地开发启动，不改动部署配置。

## 风险处理

- 如果端口已被其他非平台进程占用，启动脚本直接报错并提示先释放端口。
- 如果依赖缺失，例如 `backend/.venv` 或 `frontend/node_modules` 缺失，脚本直接给出可执行的修复提示。
- 停止脚本优先按 PID 终止进程树，避免前端残留子进程。
